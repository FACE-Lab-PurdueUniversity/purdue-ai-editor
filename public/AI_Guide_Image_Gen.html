Image Generation
& Graphic Design

Two overlapping workflows: using AI to generate original images, and using design tools to compose, layout, and communicate visually. They work best together — AI creates raw material, design tools turn it into something intentional.

How This Workflow Splits

Generation vs. composition — know which mode you're in

Track A

AI Image Generation

Prompting AI to create original images from scratch. Strong for concept art, mood reference, project visuals, and custom imagery where stock photos don't exist. Output quality depends almost entirely on prompt quality.

Tools: Claude · Midjourney · Adobe Firefly · DALL-E
Track B

Graphic Design & Composition

Assembling images, type, and layout into a finished visual — a poster, a presentation graphic, a project handout, an exhibition label. AI generates assets; design tools turn them into communication.

Tools: Canva · Adobe Express · Figma

Most projects use both: generate a background image with AI, bring it into Canva, add typography and project info, export a finished graphic. The seam between tracks is where the interesting creative decisions happen.

Core Approach

Collect references before you prompt

References before prompts

Collect 3–5 images that gesture toward what you want — not to copy them, but to identify what they have in common. Is it the color temperature? The amount of negative space? The texture? Those observations become your prompt vocabulary.

Describe the feeling, then the scene

Lead with mood and emotional register before you describe what's in the image. "Melancholic and still, as if something just left the room — a bare wooden chair near a window" is stronger than starting with the chair. The emotional frame shapes every other decision the model makes.

Iterate the prompt, not just the seed

Don't re-roll the same prompt hoping for a different result. Study what came back: what's wrong, specifically? Too bright? Wrong medium? Off mood? Change one element of the prompt, re-run, compare. Prompt iteration is design iteration.

Prompt Structure

The anatomy of a strong image prompt

Six layers — work through them in order

Subject What is in the image — person, object, place, abstract form
Medium How it's made — oil painting, digital illustration, photograph, watercolor, charcoal
Style Visual language — Art Nouveau, brutalist, isometric, cinematic, editorial
Mood Emotional register — melancholic, urgent, calm, uncanny, nostalgic, celebratory
Technical Lighting, color palette, aspect ratio, camera angle, focal length
Exclude What NOT to include — text, faces, cluttered backgrounds, specific colors
Example: "A single wooden chair in an empty theater, oil painting, Art Nouveau linework, melancholic and expectant as if a performance just ended, warm amber footlight glow, deep shadows, no people, no text."
Visual Brief — write this before generating

A short paragraph you keep open while prompting. Anchors your session and prevents drift across multiple generations.

Project: [Name — e.g., The Mirror installation] Emotional target: [What should viewers feel when they see this image?] Norman level I'm designing for: [visceral / behavioral / reflective — pick one primary] Visual tone: [3 adjectives — e.g., still, heavy, intimate] What this image is for: [background, poster, slide, exhibition panel, etc.] Constraints: [any hard requirements — color palette, no faces, specific dimensions] Reference images: [describe 1-2 images you've found that gesture toward this]

Workflow

From blank brief to finished visual asset

1

Write your Visual Brief before touching any tool

What is this image for? What should a viewer feel? Which Norman level is primary — visceral (immediate sensory hit), behavioral (how they interact with it), or reflective (what it means after)? Write 4–6 sentences. This brief goes into every AI session for this visual.

2

Collect 3–5 reference images

Search for images that gesture toward your emotional target — not to copy, but to identify what specific visual properties you're chasing. Save them in a folder. Before your first generation, describe what they share: "All of them have very little color. The light comes from one side only. There's a lot of empty space."

Those observations are your first prompt
3

Write a structured prompt using the six layers

Subject → Medium → Style → Mood → Technical → Exclusions. Start with the mood layer and work outward. If you're unsure how to describe a style, describe a reference image to Claude and ask: "What's the visual style of this? Give me 3–5 terms I could use in an image generation prompt."

4

Generate and evaluate against your Visual Brief

Run the first generation. Compare against your brief, not against a vague feeling. Write down what's wrong specifically: "Too bright — the mood I want is heavier." "The medium looks digital, not painterly." "Good composition but the color is pulling happy when I need melancholic." One issue at a time.

5

Iterate on the prompt, not the seed

Change one thing in your prompt and regenerate. Keep a running log of what you changed and what it did. After 3–5 rounds, you'll have a much clearer picture of what variables actually matter for your specific vision. Save the prompt that got closest.

6

Check rights and document before using

Before placing an AI-generated image in a final project, know: Who owns it? Can you use it in this context? Different tools have different policies — Firefly images are generally safer for broad use than Midjourney. Document your prompt and which tool generated it. In an academic or exhibition context, you should always be able to explain how an image was made.

7

Bring into Canva (or Figma) and compose

The generated image is raw material. Import it into your design tool, crop to the right aspect ratio, set typography, add any other elements. The AI generates the asset; the design tool turns it into communication. Don't skip this step — a strong image in a weak layout still reads as unfinished.

Ask Claude: "What typography would complement this image's visual tone?"
Ethics & Authorship — know before you post or present

Four questions to answer before using any AI-generated image

Training data: Was this model trained on licensed content? Midjourney and DALL-E have faced lawsuits; Adobe Firefly was trained on Adobe Stock. This doesn't make one choice automatically right or wrong — but you should know what you're using and be able to discuss it.
Authorship: Can you describe what creative decisions you made? Pressing "generate" is not enough. Your Visual Brief, your prompt construction, your evaluation and iteration — that's your creative contribution. Be able to describe it.
Transparency: In a class critique, exhibition, or public presentation — say that it's AI-generated. "I generated this image using [tool] with this prompt. Here's what I was going for and what I changed across iterations." That's the professional standard in design right now.
Energy: Image generation is computationally expensive. Generating 50 variations to find one you like has a real energy cost. Invest in prompt quality upfront so you iterate 5 times thoughtfully rather than 50 times randomly.
Prompt Patterns

How to phrase it

Pattern 01

Generating a project-specific image

Here is my Visual Brief: [Paste brief] Here are the reference images I collected: [Describe 2–3 images: what's in them, what visual properties they share] Using the reference images and the brief, write me a detailed image generation prompt using this structure: Subject / Medium / Style / Mood / Technical specs / Negative prompts Then generate the image using that prompt. After generating, tell me which layer of the prompt you think is doing the most work and which layer is weakest.
Asking Claude to critique its own prompt after generating surfaces assumptions you'd otherwise miss until iteration 4.
Pattern 02

Iterating on mood — the image is close but the feeling is wrong

Here is my current image prompt: [Paste prompt] Here is what I got: [attach image or describe] The problem is specifically: the mood is [too bright / too neutral / too aggressive / etc.] — I want [specific emotional description]. The composition and subject are mostly right. Please revise only the mood and technical layers of the prompt (don't change the subject or medium). Explain what you changed and why it should shift the feeling in the direction I want.
Isolating which prompt layer to revise keeps your other decisions stable — you're not starting over, you're adjusting one dial.
Pattern 03

Getting from generated image to Canva layout

I have a generated image I'm happy with. [Describe it briefly] I need to turn it into a [poster / exhibition label / slide graphic / handout] for [context: exhibition, class presentation, printed program]. The information it needs to carry: [title, tagline, any other text] Given the image's visual tone, what typography recommendations do you have? Suggest: — Font category (serif / sans / display) and why — Weight (bold / regular / light) and size hierarchy — Color for the type (pull from image? high contrast? something neutral?) — Where in the composition the text should live I'll be building this in Canva.
This bridges Track A (generation) to Track B (composition). The typography advice is grounded in the specific image you made — not generic advice.
Common Pitfalls

What goes wrong and why

Prompting with vibes instead of vocabulary

"Make it aesthetic" or "something cool and dark" produces generic results. AI image models respond to specific visual vocabulary — medium, style, mood, lighting — not to vague approval-seeking language.

Write your Visual Brief first. Then use the six-layer prompt structure. If you don't know what vocabulary to use, describe a reference image to Claude and ask it to extract the visual terms.

Re-rolling without changing the prompt

Generating the same prompt 10 times hoping a better version appears is random search, not design. You'll get variation in the wrong dimensions while the core problem stays fixed.

Identify exactly what's wrong with the current result. Change one layer of the prompt. Generate once. Compare. Repeat. Five intentional iterations beats fifty random ones.

Treating the generated image as finished

Almost no AI-generated image goes straight into a final project. It needs to be cropped, color-graded, composed with type, sized for its context. Dropping a raw generation onto a slide or poster shows — the edges don't match the intent.

Build the composition step into your plan from the start. The generated image is an asset, not a finished design. Always pass it through a design tool (Canva, Figma, Express) before final use.

Ignoring the ethics and authorship conversation

In a design or engineering course — and especially in a public exhibition — AI image use without acknowledgment reads as evasive. Reviewers and faculty will notice and will ask. Not having an answer is worse than the conversation itself.

Document your prompt and tool choice. Prepare a one-sentence explanation: "I generated this using [tool] with this prompt, and here's what I was trying to achieve." That's the professional standard, and it demonstrates that you made real creative decisions.

Using design tools as image generators (or vice versa)

Canva's built-in AI generation is convenient but lower quality than dedicated generators. Using Midjourney to create a poster (when it can only generate an image) means wrestling with text and layout in a tool that doesn't support it.

Use the right tool for each step: dedicated image generators (Claude, Midjourney, Firefly) for asset creation; dedicated design tools (Canva, Express, Figma) for composition. Import assets across tools rather than forcing one tool to do both jobs.
